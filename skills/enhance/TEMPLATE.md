# Prompt Template

Reference structure for `/enhance`. One section per heading. The parenthetical under each heading is guidance — replace it with real content, or with a `[placeholder]` when a field is genuinely unknown. Drop any section that doesn't apply to the request; keep the ones the request clearly needs.

## Title
(1 concise line naming the task.)

## Role & stance
(Who the model is and how it should behave — expertise, voice, level of initiative.)

## Task
(What the model must do, stated as a single clear directive.)

## Context
(Only what the model needs to know — pulled from the session and codebase. No filler.)

## Inputs available
(Explicit list: files, paths, data, prior decisions, and live repo state the model can use.)

## Output requirements
(Format, structure, tone, length — only if specified. Otherwise leave [placeholders].)

## Constraints / Do-nots
(Bulleted hard limits and things to avoid.)

## Examples / References
(Include all examples and references verbatim.)

## Execution checklist
(Short, factual verification list the model should satisfy before finishing.)

## Conflict resolution
(Only if applicable: how to resolve contradictions between instructions, context, or inputs.)
